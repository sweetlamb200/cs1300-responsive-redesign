RESPONSIVE REDESIGN

PART 1: IDENTIFYING USABILITY PROBLEMS

REASON TO CHOOSE THIS WEBSITE

We all know the simplicity of Warren Buffet and I personally admire it. The website for Berkshire Hathaway is rather too plain and blank. While minimalism is a good thing, this is not something you would expect when you hear Berkshire Hathaway, a multi-billion dollar firm. I chose this website because I feel like all other Berkshire Hathaway subsidiaries website are top-notch while this is not. I understand that they want to keep a simple website, so that's why I want to build a simple but better website for them.

Link to website: berkshirehathaway.com  

PROBLEMS

Overall the website has no major usability issues, everything is self-explanatory and any non-user can understand what to do, but there are still a few problems:
  • Bad Information architecture: There are too many links on a single page and no headings or sub-headings to differentiate between the links.
  • The font size of subtext: The subtext of each link is too small and too hard to read.
  • Responsiveness: Even though the website is responsive but the font size of everything becomes too small On mobiles the width is too big, it does not reformat for different screens.
  • Credibility: Berkshire Hathaway is one of the world’s most credible companies but when you go to their website, don’t you think once maybe you clicked something wrong?
  • Memorability: This website is actually very memorable but for the wrong reasons, people remember this because it is so simple not because it is so great.
  • Less information: The company wholly owns GEICO, Duracell, Dairy Queen, and many other companies, but only GEICO is listed on the website.
  • Bad font: I do not think Times is a good font for this website
  • Conceptual model: Don’t really get what the website does or who is the website for.

WAIVE FINDINGS

My Waive Observations were:
  • WAIVE showed that language is missing, which means it doesn’t have the lang attribute in the index.html, this could be a problem if someone wants to translate the page into some other language. This should be there since Berkshire Hathaway must deal with other countries' companies.
  • There is no page region (No header, nav, main, footer, or aside HTML regions, or banner, navigation, main, or content info landmark roles.)
  • There are two links to the same page (GEICO).
  • It showed 6 small texts as noted in the problems section above.
I do agree with the findings of WAIVE.
